# Inventory Web

Система для ведения инвентаризационного учета оборудования в разрезе компаний и сотрудников.

## Ключевые возможности

*   **Учет компаний:** Создание, редактирование и удаление информации о компаниях.
*   **Учет сотрудников:** Закрепление сотрудников за определенными компаниями.
*   **Учет оборудования:**
    *   Ведение справочника типов оборудования.
    *   Добавление, изменение и удаление единиц оборудования.
    *   Закрепление оборудования за сотрудниками.
    *   Фильтрация и поиск по каталогу.
*   **Генерация отчетов:** Создание отчетов в формате `.xlsx`.
*   **Интеграция с Telegram:** Отправка уведомлений через Telegram-бота.

## Технологический стек

*   **Бэкенд:** Python, Django
*   **База данных:** PostgreSQL
*   **Веб-сервер:** Gunicorn
*   **Инструменты для разработки:**
    *   `uv` для управления зависимостями.
    *   `ruff` для линтинга и форматирования кода.

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <repository-url>
    cd inventory_web
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```

3.  **Установите зависимости:**
    Используйте команду `make` для установки всех необходимых пакетов.
    ```bash
    make install
    ```

4.  **Настройте переменные окружения:**
    Создайте файл `.env` в корне проекта и добавьте в него необходимые переменные (например, для подключения к БД и токен для Telegram-бота).
    ```dotenv
    SECRET_KEY='your-secret-key'
    DEBUG=True
    DATABASE_URL='postgres://user:password@host:port/dbname'
    TELEGRAM_BOT_TOKEN='your-bot-token'
    ```

5.  **Примените миграции базы данных:**
    ```bash
    make migrate
    ```

6.  **Запустите сервер для разработки:**
    ```bash
    make dev
    ```
    Сервер будет доступен по адресу `http://localhost:8000`.

## Основные команды

В проекте используется `Makefile` для упрощения выполнения стандартных задач.

*   `make install`: Установить зависимости проекта.
*   `make dev`: Запустить сервер для разработки.
*   `make start-render`: Запустить production-сервер с помощью Gunicorn.
*   `make migrate`: Создать и применить миграции базы данных.
*   `make test`: Запустить тесты.
*   `make lint`: Проверить код с помощью линтера `ruff`.
*   `make format`: Отформатировать код с помощью `ruff`.

## Структура проекта

Проект построен на базе фреймворка Django и разделен на несколько приложений:

*   `inventory_web/`: Основной пакет проекта.
    *   `companies/`: Управление компаниями.
    *   `devices/`: Управление оборудованием и его типами.
    *   `employees/`: Управление сотрудниками.
    *   `users/`: Управление пользователями системы.
    *   `reprtsgen/`: Логика генерации отчетов.
    *   `telegram/`: Настройки для интеграции с Telegram.
*   `templates/`: HTML-шаблоны для интерфейса.
*   `media/reports/`: Директория для сгенерированных отчетов.
